⬛ LSB STEGANOGRAPHY
01Each pixel = 4 channels: R, G, B, A. Each channel = 8 bits.
02Only the least significant bit (bit 0) is modified. Value changes by ±1 at most.
033 bits stored per pixel (R, G, B). Alpha is untouched.
04First 32 pixels store the message length header. Remaining store payload.
05The STGNS:: signature verifies authorship during decode.
◈ XOR ENCRYPTION
01Passkey bytes XOR-combined with message bytes cyclically.
02Symmetric — same operation encodes AND decodes.
03Wrong key → non-printable output → app rejects and warns.
04Example: A(65) XOR K(75) = 10. Applying again: 10 XOR K(75) = 65 = A.

💡 BEST PRACTICES
›Always use PNG — never re-save encoded images as JPEG.
›Larger images offer more capacity and make detection harder.
›Share passkeys through a different channel than the image.
›The encoded image is visually identical to the original.
›You can hide any UTF-8 text: passwords, JSON, coordinates, code.
›XOR is not AES — for high security, encrypt externally before encoding.
